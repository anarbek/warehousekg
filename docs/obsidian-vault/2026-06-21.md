## Lessons Learned

### EF Core patterns
- **`IgnoreQueryFilters()`** is necessary for cross-tenant queries. All tenant queries in handlers and controllers must call `.IgnoreQueryFilters()` since the global filter scopes every query to the current tenant.
- **TenantId on Tenant entity**: Set to `Guid.Empty` for global entities since tenants themselves aren't scoped to another tenant. The `ApplyTenantId()` guard `if (tenantId == Guid.Empty) return;` prevents the global filter from overriding it.

### Seed data requirements
- **ItemCategory** requires `Code` (not just `Name`) and `IsActive`.
- **UnitOfMeasure** requires `Code` and `IsActive`.
- Always test seed data with actual entity validation — missing required fields cause `InternalServerError` during SaveChanges.

### Test patterns
- **`PutRawAsync`** helper needed in `WarehouseKgClient` for PUT operations — `PostRawAsync` maps to HTTP POST and returns `405 MethodNotAllowed` on PUT endpoints.
- **Capture admin username** in create-tenant tests before the create call — a second `Guid.NewGuid()` creates a different value that won't match the created user.

### Angular patterns
- **DevExtreme `DxToolbarModule`** (lowercase 'b') is the correct module name.
- **Side nav filter**: `isAdmin = data.roles.includes('Admin') || data.roles.includes('Superadmin')` shows everything to both roles. For Superadmin the "Суперадмин" item should always be visible regardless of permissions.

### Authorization design
- **Superadmin bypass in `TenantPermissionHandler`**: Essential since superadmin has no tenant scope. Without the bypass, the handler's tenant permission lookups would fail for `TenantId = Guid.Empty`.
- **Two separate policies** used: `RequireSuperadmin` for tenant management, resource-level policies for everything else. Superadmin bypasses resource policies but still needs `RequireSuperadmin` for the tenants controller.

## Lessons Learned (continued)

### Module system design
- **`enabledModules` must be in auth response** — the `my-permissions` endpoint is the central source of UI configuration. Adding `enabledModules` there allows the sidebar to filter without an extra API call.
- **String[] over comma-separated** — the API returns `["inventory"]` (array) but stores as `"inventory,crm"` (string). Parse on load, join on save. TagBox works natively with arrays.
- **Superadmin always bypasses** — module filtering should never apply to superadmin (`TenantId = Guid.Empty`).

### Form patterns
- **PUT requires all required fields** — the edit form was missing `slug`, causing `400 Bad Request`. Always include all required fields from the request DTO in the form.
- **TagBox over TextBox for multi-select** — DevExtreme TagBox with `showSelectionControls: true` provides checkbox-style multi-select with Russian labels, much better UX than comma-separated text.

### Sidebar UX
- **Collapse by default** — large sidebar groups should start collapsed (`expanded: true` removed) to avoid overwhelming the user.
- **Module-to-nav mapping** — each nav group needs a `module` field for filtering. Items without `module` (dashboard, admin) are always visible.

## Lessons Learned (continued)

### EF Core — Navigation property collection update pattern

**Problem**: `UpdatePreOrderCommandHandler` used `.Include(p => p.Lines)`, then tried to modify the lines collection via `_context.PreOrderLines.Remove(line)` + `preOrder.Lines.Clear()` + `preOrder.Lines.Add(...)`. This caused `DbUpdateConcurrencyException` ("expected to affect 1 row(s), but actually affected 0 row(s)") because EF Core's change tracker got confused when entities were both in the navigation collection and marked as Deleted.

**Fix**: Do NOT `.Include()` child collections when you plan to rebuild them. Instead:
1. Load the parent without `.Include()`
2. Query old children directly: `_context.PreOrderLines.Where(l => l.PreOrderId == request.Id).ToListAsync()`
3. Delete old children: `_context.PreOrderLines.RemoveRange(oldLines)`
4. Add new children with explicit FK: `_context.PreOrderLines.Add(new PreOrderLine { PreOrderId = request.Id, ... })`
5. Compute total from request lines, not from navigation property

This avoids all EF Core navigation property tracking conflicts.

### Flutter — `initialValue` on `DropdownButtonFormField`

Flutter 3.33+ **deprecated** `value` on `DropdownButtonFormField` in favor of `initialValue`. Always use `initialValue`.

### Flutter — `FocusScope.of(context).unfocus()` for autocomplete

After selecting an item in `Autocomplete` or after clicking "Добавить", call `FocusScope.of(context).unfocus()` to dismiss both the keyboard and the dropdown overlay.

### Flutter — `TextFormField` needs `onChanged` + `setState` for live recalculation

If a `TextFormField`'s value is used in a computed expression elsewhere in the widget tree, it MUST have an `onChanged` callback that calls `setState`. Without it, changes to the field don't trigger a rebuild, and the computed values stay stale. The discount field had `onChanged` but the quantity field didn't — so the sum only updated when discount was changed.

### Flutter — Modal layout: `Column` with `Expanded` for scrollable list

When building a modal with input fields at top and a scrollable list below, use `Column(children: [fixed_fields..., Expanded(child: ListView(...))])` instead of a single `ListView` wrapping everything. This keeps the input fields always visible.

### Angular — `dx-number-box` select-all on focus

DevExtreme's `dx-number-box` doesn't natively support select-all-on-focus. Use `(onFocusIn)="onNumberFocusIn($event)"` with:
```typescript
onNumberFocusIn(e: any) {
  setTimeout(() => {
    const input = e.element?.querySelector('input');
    if (input) input.select();
  });
}
```
The `setTimeout` is needed because DevExtreme's focus handler runs before the native input is ready.

### Angular — Auto-price from inventory on stock select

Use `(onValueChanged)` on the `dx-select-box` in the grid cell template. The event fires both on user selection and on initial data binding (edit mode). Look up `unitPrice` from the loaded `inventoryItems[]` array and only override if current price is 0.

### API — `UpdatePreOrderCommand` requires `id` in body

The `PreOrdersController.Update` checks `if (id != command.Id) return BadRequest()`. The PUT body must include `"id": "<guid>"` matching the URL parameter, or the request returns 400. This applies to all commands that use `Guid Id` as a record parameter.
