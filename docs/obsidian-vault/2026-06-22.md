## Application Layer

### CQRS Commands
- **`CreateInvoiceCommand`** — Creates invoice from customer/warehouse/lines. Generates sequential number `INV-{YEAR}-{NNNN}`. Computes `TotalAmount`, `TaxAmount`, per-line totals.
- **`IssueInvoiceCommand`** — Transitions Draft → Issued.
- **`PrintInvoiceCommand`** — Transitions Issued/Printed → Printed. Records `PrintedAtUtc`.
- **`SignInvoiceCommand`** — Transitions Issued/Printed → Signed. Accepts optional `SignedByName`.
- **`CancelInvoiceCommand`** — Transitions Issued/Printed → Cancelled. Cannot cancel already Signed invoices.
- **`DeleteInvoiceCommand`** — Deletes invoice and all its lines.

### CQRS Queries
- **`GetInvoicesQuery`** — Paginated list with optional filters: Status, CustomerId, WarehouseId, DateFrom, DateTo. Returns `InvoiceSummaryDto` with resolved CustomerName, WarehouseName, SalesOrderNumber.
- **`GetInvoiceByIdQuery`** — Full detail with resolved names (Customer, Warehouse, Supplier, SalesOrder) and lines with InventoryItem SKU.

### DTOs
- **`InvoiceDto`** — Full detail: all fields + `Lines` collection + `CreatedAt`.
- **`InvoiceLineDto`** — Line detail: InventoryItemId, InventoryItemName (SKU), Quantity, UnitPrice, LineTotal, TaxRate, TaxAmount.
- **`InvoiceSummaryDto`** — List view: Id, Number, Type, Status, CustomerId, CustomerName, WarehouseId, WarehouseName, IssuedAtUtc, CreatedAt, TotalAmount, Currency, SalesOrderId, SalesOrderNumber, LineCount.

## Lessons Learned

### Tax rate semantics
- User enters `12` meaning **12%**, not `0.12`. Backend and frontend must **divide by 100**: `TaxAmount = lineTotal * taxRate / 100`.
- Fixed in 3 places in `CreateInvoice.cs`, update handler, frontend `recalcTotals()`, and PDF display (`{l.TaxRate}%` not `{l.TaxRate:P1}`).

### EF Core — Update child collections safely
- Do NOT `.Include()` child collections when rebuilding them. Query old children directly via `.Where(l => l.ParentId == request.Id)`, delete with `RemoveRange`, add new with explicit FK. Avoids `DbUpdateConcurrencyException`.
- Two separate `SaveChangesAsync` calls: first for old line deletion, second for new line insertion + parent update.

### DevExtreme `dx-data-grid` cellTemplate
- Cell templates can reference `c.data.totalAmount + c.data.taxAmount` for computed columns.
- The `number` pipe requires `DecimalPipe` in the component's `imports` array.
- `dxo-summary` with `dxi-total-item` shows column footer sums; use `[customizeText]` for formatting.

### DevExtreme widget interaction via browser tools
- DevExtreme components use shadow DOM + overlays; `click_element` often times out due to watermark overlay.
- Workaround: inject auth tokens directly into `localStorage` (`wkg.accessToken`, `wkg.tenantId`, `wkg.user`, etc.) via `page.evaluate()`, then `page.goto()` to target page.
- Login form can be submitted via `page.locator('input[placeholder="..."]').fill()` + keyboard `Enter`.

### Angular reactive forms — live totals
- `computed()` signals don't react to `FormControl` value changes directly.
- Use `form.valueChanges.subscribe(() => this.recalcTotals())` to update signals reactively.
- `[(value)]` two-way binding on DevExtreme widgets bypasses `setValue()` — use `[value]` + `(onValueChanged)="control.setValue($event.value)"` instead.

### Invoice list grand total
- `InvoiceSummaryDto` originally had only `TotalAmount` (subtotal). Added `TaxAmount` so list can show grand total `totalAmount + taxAmount`.
- Backend: Added `TaxAmount = i.TaxAmount` to handler's `.Select()`.
- Frontend: Added `taxAmount` to `InvoiceSummary` interface, changed list column from `dataField` to `cellTemplate`.

### Angular — DecimalPipe in standalone components
- Using `| number:'1.2-2'` in templates requires `DecimalPipe` in the component's `imports` array. Missing import causes `NG8004: No pipe found with name 'number'`.

### Form sum row alignment
- Sum row must use same CSS class (`form-lines__computed`) as data row computed cells to ensure identical column widths.
- Sum row must have `> * { flex: 1; min-width: 0 }` matching `__row` flex behavior.
- Empty spacer columns needed for non-numeric columns to maintain grid alignment.
