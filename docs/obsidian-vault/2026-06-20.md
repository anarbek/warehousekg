## Backend — Bug Fixes

### Auto-ship inventory deduction
- **Bug**: `CompleteDeliveryStopCommandHandler` set `Status = Shipped` but never deducted `QuantityOnHand`. Inventory was only deducted by `ShipSalesOrderCommandHandler` (manual ship).
- **Fix**: Added `.Include(so => so.Lines)` and inventory deduction loop identical to `ShipSalesOrderCommandHandler` in the auto-ship path of `CompleteDeliveryStopCommandHandler`.
- **File**: `src/WarehouseKG.Application/Features/Dispatching/Stops/Commands/StopWorkflow.cs`

### SalesOrder.WarehouseId required for movement history
- **Bug**: Sales orders created without `WarehouseId` never appeared in inventory movement history. The `ItemMovementHistoryQueries` filters by `SalesOrder.WarehouseId == selectedWarehouseId`, and NULL never matches.
- **Fix**: Set `WarehouseId` on existing test sales orders. For prevention, the frontend should require WarehouseId when creating sales orders.
- **Historical fix**: Updated `sales_orders.WarehouseId` for SO-TEST-001 and SO-TEST-002 from NULL to depo 1.

### TenantId Guid.Empty guard
- **Bug**: `ApplyTenantId()` in `WarehouseKgDbContext` would silently set `TenantId = Guid.Empty` when `CurrentTenantId` was empty.
- **Fix**: Added `if (tenantId == Guid.Empty) return;` guard.
- **File**: `src/WarehouseKG.Infrastructure/Persistence/WarehouseKgDbContext.cs`

## Flutter Testing Learnings

### JWT & Claims
- Driver features need `employee_id` claim in JWT. If user was created before linking to Employee record, stored JWT is stale — must logout and re-login.
- Backend restart invalidates all JWTs → logout/re-login on device required.
- `flutter_secure_storage` persists JWT across app restarts; Dio interceptor attaches it to every request.

### Emulator Connectivity
- Android emulator uses `10.0.2.2` to reach host machine (not `localhost`).
- Backend must listen on `0.0.0.0` (not just `localhost`) for emulator to connect.

### Hot Reload vs Hot Restart
- `r` = hot reload: preserves app state, good for UI tweaks.
- `R` = hot restart: clears Riverpod state, re-initializes providers. Use after backend fixes that change data.
- After `employee_id` linkage change: need full app restart (not just hot restart) — the JWT is stored and hot restart doesn't re-login.
